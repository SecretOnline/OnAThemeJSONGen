On A Theme JSON Generator

Making texture variations in Minecraft is annoying. It's a lot of tedious work, which includes copying and pasting files, only to edit a couple of things, then repeat it.

I made this to make that process a little easier. The JSON generator handles a lot of this for you, with plans to be able to do a most of the process for you.

Usage

Upon opening the program, it should look something like the screenshot below:

To add and remove block from the pack, you need to use the "Blocks" menu. Click the option to add a new block.

A new window will pop up, and will ask for a block name. For this example, I'm going to use a rose bush. As you can see, I've filled in double_rose as the Block Name. This must be the name that is used by the game. Once you've done that, click "Add" to add an attribute of this block.

Yet another window will appear. By default, the attributes are "normal".

As we're working with a two block tall block, we need an attribute for both the top and the bottom. I'm going to make the top block invisible, and have both halves of the texture determined by the bottom. If you've played with variants with blocks that take up more than one block (e.g. beds), then you'll know why I am doing this.

Included with every pack made by On A Theme are a few models. One of those is none. It creates an invisible block, but the hitbox remains. I'm going to overwrite the model name, otherwise it will conflict when we come to do the bottom half.

The next part is to define the textures for the model to use. While none has no visible texture, it still has particles when broken. To tell the model to do this, we add a reference of particle, and in our case, a texture of double_plant_rose_top.

You'll now see it in the list of textures. Since this is the only thing we need to define for the upper half of the rose bush, I'm going to click "Create".

You will now see the half=upper in the list of attributes. Now we need to create the lower one.

Add another attribute, this time with the parent model being double_cross. This is another model that's bundled with this program. It allows both halves of the plant to be consistent when doing variations. I've also put a number into the number of variants. We'll see the effect of this later. Since you've seen me do it above, I went ahead and put in the textures.

Now that we've created both halves of our block, let's see what will be created.

Back on the original window. You can select the block, attribute, and model to change the weighting and override the model name. More options will be made available as they are added. Use the number selector to change the weight. Higher values mean they are more likely to be picked. If you want to add more blocks, just follow the same steps as above.

The final thing you'll want to do is generate the files. On the File menu, select "Create UnZipped". This will create a folder named "OaT" in the directory the executable is in. Inside that will be all the files, arranged in folders like those in a resource pack. Just copy and paste, and you're good to go.

Should you wish to improve the program, the source code is available on GitHub. Thanks for trying it out, and thank you very much for actually bothering to read the instructions.
